# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | ✅ Active development |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue
in Glacier.Polaris, please report it by emailing ian.cowley@example.com.

Please **do not** report security vulnerabilities through public GitHub issues.

We will acknowledge receipt of your report within 48 hours and provide a
timeline for a fix and disclosure. We appreciate your responsible disclosure.

## Scope

Security issues include, but are not limited to:

- Buffer overflows or memory safety issues
- Code injection or remote code execution
- Denial of service via algorithmic complexity attacks
- Exposure of sensitive data through serialization/deserialization
- Unsafe use of `unsafe` code blocks

## Preferred Languages

We prefer all communications to be in English.
